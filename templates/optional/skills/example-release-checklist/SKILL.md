# Release checklist

## Purpose
Run a repeatable pre-release checklist for a human-triggered release task.

## When to use
- A release task is assigned in `TASKS.yaml`.
- The repo wants a consistent final verification pass.

## Steps
1. Read the assigned task and release notes.
2. Run the repo's verify command.
3. Confirm any docs or changelog updates.
4. Record the outcome in the task report.
