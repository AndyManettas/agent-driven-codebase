# Code review follow-up

## Purpose
Apply a repeatable workflow for addressing actionable code review feedback on an assigned task.

## When to use
- A human assigns review comments for a specific task.
- The repo wants a consistent fix-and-verify flow.

## Steps
1. Read `AGENTS.md` and the assigned task entry.
2. Read the referenced review feedback.
3. Make only task-scoped changes.
4. Run targeted verification and record the result in the task report.
